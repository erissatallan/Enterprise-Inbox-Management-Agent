## Problem

Operations teams handling shared inbox traffic still rely on manual triage and manual routing. The result is slower response time, inconsistent execution, and weak auditability.

## What I built

A deterministic, policy-gated AI workflow pipeline:

- Ingest inbound messages
- Normalize unstructured text
- Predict intent, priority, and actions
- Apply policy (`AUTO_EXECUTE`, `REQUIRE_REVIEW`, `DENY`)
- Execute low-risk actions
- Log complete decision and action trace

## Business impact (baseline)

- Intent accuracy: `91.67%`
- Priority accuracy: `100%`
- Action correctness: `100%`
- Policy accuracy: `91.67%`
- Auto-execute rate: `41.67%`
- Review rate: `58.33%`

## Architecture

Inbound events -> Normalization -> Decision Engine -> Policy Engine -> Tool Executor / Review Queue -> Audit + Reports
